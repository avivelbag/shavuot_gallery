# One Rule, All Interpretation

The Talmud Yerushalmi (Peah 2:6) records that two Torahs were given at Sinai: the Written Torah and the Oral Torah. Pirkei Avot opens with the same teaching: *Moshe kibel Torah miSinai umsarah l'Yehoshua* — "Moses received Torah at Sinai and transmitted it to Joshua, Joshua to the Elders, the Elders to the Prophets, and the Prophets transmitted it to the Men of the Great Assembly" (Avot 1:1). From the first moment, Torah was never only a text. It was a text plus the unwritten tradition of how to read it.

## The Written and the Oral

The Written Torah is finite: the Masoretes counted 304,805 letters across the Five Books of Moses, and that count has not changed in a thousand years. The text is sealed, fixed, guarded by tradition with a precision that scribes still honor today — a sefer Torah with a single letter missing is invalid, set aside until corrected. This is Torah she-bikhsav: the Written Torah.

The Oral Torah is something else entirely. Each generation of rabbis applies the fixed rules to new questions and produces new rulings that could not have been predicted in advance. The Mishnah, the Talmud, the Rishonim, the Acharonim, the contemporary responsa literature — every decade adds another layer. Rabbi Akiva, said the Midrash, was able to derive mountains of law from the decorative crowns on the letters of the Torah. The Written Torah is the seed; the Oral Torah is an open and growing tree.

## Rule 110

In 1983 Stephen Wolfram began cataloguing all 256 elementary cellular automata — systems where each cell in a 1D row updates its state based on its own value and the values of its two neighbors. Every possible such rule maps eight 3-cell neighborhoods to a 0 or 1. The entire rule fits in a single byte. Rule 110 is encoded as binary 01101110.

In 2004, Matthew Cook (working in Wolfram's group) published a proof that Rule 110 is computationally universal: given the right initial configuration, a Rule 110 grid can simulate any Turing machine, and therefore any computation that any computer can perform. The full power of all software ever written is latent in one byte, eight entries, one rule applied row by row.

The non-obvious insight is structural. Rule 110 proves that Turing-completeness — the capacity to compute anything — does not require a large or complex rule set. It requires only a rule set rich enough that, when applied repeatedly to new input, it produces output that cannot be predicted without running the rule. The results are *undecidable in advance*. You must simply run it and see what emerges.

This is exactly the structure of halakha. The Shulchan Aruch encodes a fixed rule set. But when a new question arrives — a technology the ancients could not have imagined, a situation the Talmud never addressed — the posek applies those fixed rules to the new case and produces a ruling that could not have been read off from the original text. You must run the rule.

## The Piece

What you see here is Rule 110 operating from a single seed: one ON cell at the center of the first row, everything else OFF. This is Torah she-bikhsav — a single, fixed starting condition. Row by row, the rule applies. Triangular structures emerge, irregular lacunae, glider-like patterns that propagate through the tapestry without ever settling into a regular period. The bottom of the screen is the present; the top is the past, scrolling away as each new generation pushes history upward.

The label *dor l'dor* — דּוֹר לְדוֹר, "generation to generation" — is the ancient formula for how the Oral Torah travels: not printed but transmitted, not static but alive, one generation handing the ever-growing tradition to the next. The rule is one. The output is inexhaustible.
